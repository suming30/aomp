# AutoOps

## Repository Structure

- `aomp-admin`: Frontend admin application.
- `aomp-server`: Backend service and server-side tests.
- `docs/product`: Product requirements and feature definitions.
- `docs/architecture`: Architecture notes and technical design docs.
- `docs/testing`: Integration plans, test cases, and acceptance summaries.
- `docs/progress`: Project progress records and milestone notes.
- `artifacts/screenshots`: Test screenshots and visual verification output.
- `artifacts/reports`: Generated reports and test result summaries.
- `artifacts/imports`: Imported sample data and spreadsheet assets.

## Working Rules

- Keep frontend code changes inside `aomp-admin`.
- Keep backend code changes inside `aomp-server`.
- Put new requirement or acceptance docs under `docs/product` or `docs/testing` instead of the repo root.
- Put screenshots, reports, and import files under `artifacts` instead of the repo root.
- Keep `README.md`, `task_plan.md`, and `progress.md` at the repository root for quick access.
