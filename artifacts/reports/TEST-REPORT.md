## 三、缺陷问题清单

### 缺陷 #001: 脚本仓库列表页面渲染失败

| 属性 | 内容 |
|------|------|
| **缺陷编号** | DEF-001 |
| **所属模块** | 脚本仓库 |
| **严重级别** | **高 (High)** |
| **复现步骤** | 1. 使用admin账号登录系统<br>2. 点击左侧导航"脚本中心"<br>3. 或直接访问 http://localhost:5173/scripts |
| **实际结果** | 页面完全黑屏/空白，`document.body.innerText`为空字符串，无交互元素(`no interactive elements`) |
| **预期结果** | 应显示脚本列表页面，包含新增脚本按钮、搜索框、类型筛选、脚本卡片列表 |
| **影响范围** | 脚本仓库核心功能完全不可用，影响脚本创建/编辑/执行/版本管理等全流程 |
| **根因分析** | 可能原因：①API请求失败导致v-loading永久加载；②Vue组件挂载异常；③样式z-index遮挡；④路由守卫拦截未正确处理 |
| **修复建议** | 1. 检查浏览器Console错误日志<br>2. 验证/scripts API接口是否正常响应<br>3. 检查ScriptListView.vue的onMounted数据加载逻辑<br>4. 确认CSS样式无overflow:hidden或z-index问题 |

### 缺陷 #002: 任务执行配置页面权限拦截

| 属性 | 内容 |
|------|------|
| **缺陷编号** | DEF-002 |
| **所属模块** | 任务执行引擎 |
| **严重级别** | **高 (High)** |
| **复现步骤** | 1. 使用admin账号登录系统<br>2. 点击左侧导航"任务执行"<br>3. 或直接访问 http://localhost:5173/tasks/config |
| **实际结果** | 页面顶部显示红色错误横幅"No permission to access"，内容区域可能部分或完全不可用 |
| **预期结果** | admin作为超级管理员应拥有所有模块的完整访问权限，任务配置页面应正常加载 |
| **影响范围** | 任务执行核心功能不可用，影响任务创建/配置/执行/监控全流程 |
| **根因分析** | 可能原因：①路由meta.permission配置与admin权限不匹配；②后端接口返回403但前端未正确处理；③JWT Token中permissions字段缺少task相关权限标识 |
| **修复建议** | 1. 检查router/index.js中tasks路由的meta.permission配置<br>2. 确认admin角色的permissions数组包含task相关权限<br>3. 检查后端SecurityConfig是否对/task路径有额外限制<br>4. 验证JwtAuthenticationFilter对admin用户的权限判断逻辑 |

### 已修复问题 (本次测试过程中)

| 问题 | 描述 | 修复方案 | 状态 |
|------|------|----------|------|
| FIX-001 | NotificationServiceImpl IncompatibleClassChangeError | 系统JAVA_HOME指向JDK13(2020)而非项目要求的JDK21，导致编译产物类文件格式不兼容 | 设置JAVA_HOME=C:\jdk-21.0.7后mvn clean compile重新编译 | ✅ 已修复 |
| FIX-002 | 用户管理页面导航缺失 | user.js store缺少isAdmin()方法，导致SidebarNav无法识别admin角色 | 在useUserStore中添加isAdmin()函数并export | ✅ 已修复 |
| FIX-003 | 后端端口8181占用 | 旧Java进程(PID 34528)占用端口且权限较高无法终止 | 旧进程仍在运行提供服务，新编译的class文件需重启后端生效 | ⚠️ 需手动重启 |
