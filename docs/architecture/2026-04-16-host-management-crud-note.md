# 主机管理 CRUD 补全说明

日期：2026-04-16

## 背景

主机管理页面已经接入了部分接口方法，但页面交互仍存在以下问题：

- 列表字段与后端 `HostInfoVO` 字段不一致，导致列表展示异常。
- 新增表单字段与后端 DTO 不一致，提交后无法稳定完成创建。
- 页面缺少编辑入口，删除与连通性检测仅完成了部分占位行为。
- 当前联调环境下，主机演示数据不足，不利于快速验证增删改查流程。

## 本次设计

本次补全聚焦 `aomp-admin` 的主机管理页面，采用“真实接口优先，演示数据可切换”的实现方式：

1. 保留现有后端主机接口作为真实数据来源。
2. 在前端增加主机演示数据存储与 CRUD 模拟能力，便于无后端或空数据时继续验证页面行为。
3. 主机页支持以下闭环：
   - 列表查询、状态筛选、分页
   - 新增主机
   - 编辑主机
   - 删除主机
   - 单个/批量连通性检测
   - 一键导入虚拟数据
4. 演示模式下的数据持久化到浏览器本地存储，方便重复验证。

## 边界

- 本次不扩展新的后端表结构。
- 不引入新的前端依赖。
- 仅调整主机管理页与相关前端辅助模块，不做无关页面重构。

## 验证方式

- 前端执行 `npm run build`
- 通过演示数据存储模块执行一轮增删改查脚本验证
