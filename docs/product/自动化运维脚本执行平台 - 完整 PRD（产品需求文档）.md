## 1. 项目概述

### 1.1 项目背景

传统运维模式依赖人工SSH登录服务器执行脚本，存在效率低下、脚本管理分散、操作无审计、风险不可控等痛点，导致运维成本高、故障追溯难、新人上手慢。为解决上述问题，打造标准化、自动化、安全可控的运维工具，提升运维效率，降低操作风险，特启动本自动化运维脚本执行平台项目。

### 1.2 项目目标

1. 核心目标：实现“脚本统一管理、批量下发执行、实时日志查看、操作全程可追溯”，替代人工SSH操作，提升多服务器脚本执行效率。
2. 体验目标：通过便捷交互、智能提示、自动化填充，降低运维操作成本，让普通运维人员可快速上手，高级运维人员可高效完成批量操作。
3. 安全目标：通过精细化权限控制、危险操作拦截、操作审计，确保生产环境运维操作安全可控，避免误操作和恶意操作。
4. 扩展目标：支持自定义巡检项、脚本模板、外部通知联动，适配不同业务场景的运维需求。

### 1.3 项目范围

#### 1.3.1 纳入范围

- 资产中心：主机清单、分组管理、标签管理、连接配置、连通性巡检
- 脚本管理：脚本仓库、版本控制、权限管理、标签管理、草稿保存
- 任务执行：任务定义、批量执行、实时日志、执行控制（暂停/终止/重试）
- 一键巡检：默认巡检项、自定义巡检项、巡检报告生成与导出
- 日志审计：操作日志、执行日志、审计追溯、日志管理
- 安全权限：RBAC权限模型、危险操作拦截、临时授权、双人审批
- 通知联动：多渠道通知、@提醒、Webhook回调
- 界面交互：响应式布局、暗色/亮色模式、快捷操作、引导提示

#### 1.3.2 排除范围

- 容器编排（如K8s管理）、云资源（如AWS/阿里云）直接管理
- 实时监控告警系统（可通过Webhook对接外部监控）
- 服务器硬件故障检测（仅通过脚本间接获取硬件相关信息）
- 跨平台操作系统深度定制（仅支持主流Linux/Windows系统）

### 1.4 产品定位

轻量、高效、安全的自动化运维工具，聚焦“脚本批量执行与管理”核心场景，适配中小规模运维团队（10-100人），无需复杂部署，开箱即用，可快速替代人工SSH操作，降低运维门槛。

## 2. 术语定义

| 术语        | 定义                                          |
| --------- | ------------------------------------------- |
| 主机        | 需进行运维操作的服务器（物理机/虚拟机），以IP地址为唯一标识             |
| 主机组       | 对主机进行分类的集合，分为静态组（手动维护）和动态组（自动归类）            |
| 脚本        | 可在服务器上执行的代码文件，支持Shell、Python、PowerShell三种类型 |
| 任务        | 定义“脚本+目标主机+执行参数”的操作单元，是脚本执行的载体              |
| 执行引擎      | 平台核心组件，负责建立SSH连接、下发脚本、获取执行结果并推送日志           |
| 巡检        | 通过预设脚本，批量检查服务器健康状态（如CPU、内存、磁盘等），生成报告        |
| 权限矩阵      | 基于RBAC模型，定义不同角色可操作的功能和数据范围                  |
| Dry Run模式 | 模拟脚本执行流程，仅检查语法、参数和依赖，不实际在服务器上运行脚本           |
| 静默执行      | 后台执行任务，不弹出实时日志窗口，仅记录执行结果，适合高频巡检             |
| 临时授权      | 普通用户申请的、限时有效的执行权限，用于临时操作高权限资源               |

# 3. 用户角色与权限矩阵（续）
## 3.3 用户管理模块
### 3.3.1 功能描述
对平台用户进行全生命周期管理，包括用户新增、编辑、禁用/启用、密码重置、信息查询及个人信息维护，实现平台使用者统一身份管理，配合角色体系实现权限分发与回收。

### 3.3.2 字段说明
| 字段名称 | 字段类型 | 必填项 | 默认值 | 字段说明 | 约束条件 |
|---|---|---|---|---|---|
| 用户ID | 字符串 | 是 | 系统自动生成 | 用户唯一标识，格式：USER-年月日-随机6位 | 不可修改，全局唯一 |
| 登录账号 | 字符串 | 是 | 无 | 用户登录平台账号 | 长度4-20位，字母/数字/下划线，全局唯一 |
| 用户名 | 字符串 | 是 | 无 | 平台内显示名称 | 长度2-20字符，支持中英文 |
| 邮箱 | 字符串 | 否 | 无 | 用户通知、找回密码邮箱 | 符合邮箱格式，全局唯一 |
| 所属角色 | 列表 | 是 | 普通运维 | 关联角色列表，一个用户可绑定多个角色 | 至少选择1个角色 |
| 账号状态 | 枚举 | 是 | 启用 | 启用/禁用/锁定 | 禁用/锁定用户无法登录 |
| 密码 | 字符串 | 是 | 系统随机生成 | 登录密码，加密存储 | 明文8位以上，含大小写+数字+特殊字符 |
| 强制改密标记 | 布尔 | 是 | true | 首次登录/密码重置后是否强制修改密码 | true=强制修改，false=不强制 |
| 最后登录IP | 字符串 | 否 | 无 | 最近一次登录IP | 系统自动记录，不可编辑 |
| 最后登录时间 | 日期时间 | 否 | 无 | 最近一次登录时间 | 系统自动记录，不可编辑 |
| 负责人 | 字符串 | 否 | 无 | 该用户业务归属负责人 | 关联平台用户 |
| 备注 | 文本 | 否 | 无 | 用户说明、岗位等补充信息 | 长度≤200字符 |
| 创建人 | 字符串 | 是 | 当前管理员 | 创建该用户的管理员 | 不可修改 |
| 创建时间 | 日期时间 | 是 | 系统当前时间 | 用户创建时间 | 不可修改 |
| 更新时间 | 日期时间 | 是 | 系统当前时间 | 信息最后修改时间 | 系统自动更新 |

### 3.3.3 交互说明
1. 用户列表
    - 支持按登录账号、用户名、角色、账号状态筛选搜索；
    - 列表展示：用户头像、登录账号、用户名、角色、状态、最后登录时间；
    - 禁用/锁定用户行置灰处理，便于视觉区分。
2. 新增用户
    - 管理员点击「新增用户」弹出表单，填写账号、姓名、邮箱、角色等信息；
    - 密码支持手动设置或系统随机生成，生成后仅展示一次；
    - 提交后自动发送账号通知（站内信/邮件）。
3. 编辑用户
    - 支持修改用户名、邮箱、角色、状态、备注；
    - 登录账号不可修改；
    - 角色变更后实时生效。
4. 密码重置
    - 管理员可对任意用户执行重置密码操作；
    - 重置后强制改密标记置为true，用户下次登录必须修改密码。
5. 批量操作
    - 支持批量启用、批量禁用、批量调整角色；
    - 操作前二次确认，防止误操作。
6. 个人信息维护
    - 用户可在个人中心修改用户名、邮箱、头像；
    - 支持自主修改登录密码，修改后当前会话不退出。

### 3.3.4 业务规则
1. 唯一性规则：登录账号、邮箱全局唯一，不允许重复。
2. 状态规则
    - 启用：正常登录、操作；
    - 禁用：禁止登录，已登录会话强制下线；
    - 锁定：连续登录失败达到阈值自动锁定，管理员可手动解锁。
3. 密码规则
    - 密码采用单向加密存储，任何场景不展示、不传输明文；
    - 密码有效期90天，到期前7天开始提醒；
    - 不允许使用近3次历史密码。
4. 登录保护
    - 连续5次登录失败，账号自动锁定15分钟；
    - 无操作30分钟自动登出；
    - 所有登录行为记录日志，包含IP、设备、时间、结果。
5. 删除规则
    - 平台不支持物理删除用户，仅支持禁用；
    - 用户禁用后，其历史操作、任务、脚本、日志全部保留，用于审计追溯。

## 3.4 角色与权限管理模块
### 3.4.1 功能描述
基于RBAC权限模型，实现角色定义、权限绑定、权限分配与回收，支持系统内置角色与自定义角色并存，实现功能权限与数据权限统一管控。

### 3.4.2 字段说明
| 字段名称 | 字段类型 | 必填项 | 默认值 | 字段说明 | 约束条件 |
|---|---|---|---|---|---|
| 角色ID | 字符串 | 是 | 系统自动生成 | 角色唯一标识 | 不可修改 |
| 角色名称 | 字符串 | 是 | 无 | 角色名称，如：生产运维、审计员 | 长度2-20字符，唯一 |
| 角色描述 | 文本 | 否 | 无 | 角色职责说明 | 长度≤100字符 |
| 角色类型 | 枚举 | 是 | 自定义 | 系统内置/自定义 | 系统内置角色不可删除 |
| 功能权限 | 列表 | 是 | 空 | 菜单查看、按钮操作权限集合 | 以权限标识符绑定 |
| 数据权限 | 枚举 | 是 | 个人 | 个人/同团队/全部 | 控制主机、脚本、任务可见范围 |
| 状态 | 枚举 | 是 | 启用 | 启用/禁用 | 禁用后关联用户自动失去权限 |
| 创建人 | 字符串 | 是 | 当前管理员 | 不可修改 |
| 创建时间 | 日期时间 | 是 | 系统当前时间 | 不可修改 |
| 更新时间 | 日期时间 | 是 | 系统当前时间 | 系统自动更新 |

### 3.4.3 交互说明
1. 角色列表
    - 展示角色名称、类型、状态、关联用户数、描述；
    - 系统内置角色标注明显标识，不显示删除按钮。
2. 角色配置
    - 采用树形权限勾选面板，左侧菜单、右侧按钮权限；
    - 支持全选/反选/展开折叠；
    - 数据权限单独下拉选择。
3. 复制角色
    - 支持以现有角色为模板复制，快速创建相似权限角色。
4. 禁用角色
    - 角色禁用后，所有关联用户立即失去该角色权限。

### 3.4.4 业务规则
1. 系统内置角色（普通运维、高级运维、管理员、开发人员）不可删除、不可重命名，可调整权限。
2. 一个用户可拥有多个角色，权限取并集。
3. 功能权限精确到按钮级别，如：查看/新增/编辑/删除/执行/审批/导出。
4. 数据权限规则
    - 个人：仅可查看自己创建/负责的资源；
    - 同团队：可查看同团队成员资源；
    - 全部：可查看平台所有资源。
5. 权限变更实时生效，无需用户重新登录。
6. 禁止任何用户将自身权限提升为超级管理员。

## 3.5 高级安全权限控制
### 3.5.1 临时授权管理
### 3.5.1.1 功能描述
用于满足应急场景下的短期权限提升，支持按用户、时间、资源范围授权，到期自动回收，实现最小权限与应急弹性的平衡。

### 3.5.1.2 字段说明
| 字段名称 | 字段类型 | 必填 | 说明 |
|---|---|---|---|
| 授权ID | 字符串 | 是 | 唯一标识 |
| 授权用户 | 字符串 | 是 | 被授权人 |
| 授权角色 | 列表 | 是 | 临时授予的角色 |
| 授权资源 | 列表 | 否 | 限定主机/主机组 |
| 生效时间 | 日期时间 | 是 | 授权开始时间 |
| 失效时间 | 日期时间 | 是 | 授权结束时间 |
| 授权原因 | 文本 | 是 | 应急事由 |
| 授权人 | 字符串 | 是 | 审批/授权人 |
| 状态 | 枚举 | 是 | 待生效/生效中/已过期/已撤销 |

### 3.5.1.3 业务规则
1. 临时授权仅管理员可发起，授权记录永久留存审计；
2. 到期自动回收权限，不影响用户原有角色；
3. 支持提前手动撤销授权；
4. 生产环境临时授权最长不超过24小时。

### 3.5.2 高危操作双人审批
### 3.5.2.1 功能描述
生产环境高危脚本、高危命令、批量重启、数据清除等操作，必须经过第二名管理员审批方可执行，防止单人误操作。

### 3.5.2.2 业务规则
1. 触发审批场景
    - 生产环境执行含黑名单命令的脚本；
    - 一次性操作≥10台生产主机；
    - 重启、关机、格式化相关操作。
2. 审批流程：提交任务→进入审批池→管理员审批→通过执行/驳回终止；
3. 审批时效为2小时，超时自动关闭；
4. 审批人与提交人不能为同一用户；
5. 所有审批记录不可篡改，纳入审计日志。

### 3.5.3 危险命令拦截规则
1. 系统内置黑名单命令：`rm -rf /`、`mkfs`、`dd`、`halt`、`reboot`、`shutdown` 等；
2. 脚本在保存、执行时自动扫描危险命令；
3. 拦截策略
    - 一级拦截：直接禁止执行；
    - 二级拦截：强制双人审批；
    - 三级提醒：强提醒二次确认。
4. 生产环境强制开启拦截，测试环境可由管理员配置策略。

## 3.6 权限相关审计规则
1. 所有权限相关操作必须落审计日志，包括：用户新增/禁用、角色修改、权限变更、临时授权、审批、密码重置；
2. 日志内容包含：操作人、操作对象、操作内容、IP、时间、结果；
3. 权限变更日志至少保存180天，不可删除、不可修改；
4. 普通用户仅可查看自己的权限与操作记录，管理员可查看全量审计日志；
5. 任何越权访问行为立即拦截，并记录高危日志。

## 4. 核心功能模块（含字段/交互/业务规则）

### 4.1 资产中心（主机管理）

#### 4.1.1 主机清单管理

##### 4.1.1.1 功能描述

管理所有运维服务器的基础信息、连接信息，支持添加、编辑、删除、查询、批量导入/导出，实时展示主机在线状态。

   - 支持按IP地址、主机别名、主机名模糊搜索，输入关键词实时联想匹配，搜索结果即时刷新；

   - 主机列表支持分页，默认每页20条，可切换10/20/50/100条/页；

2. 单个添加交互：

   - 输入IP地址后，系统自动校验格式，格式错误提示“请输入正确的IPV4地址”；

   - 回收站页面支持按删除时间筛选，点击“恢复”按钮，主机状态恢复为“离线”（需重新测试连通性）；点击“彻底删除”，主机信息永久删除，不可恢复。

##### 4.1.1.4 业务规则

1. 唯一性规则：IP地址+SSH端口组合唯一，不可重复添加（避免同一服务器重复录入）；
2. 状态更新规则：系统每30分钟自动检测所有主机SSH连通性，更新主机状态；用户手动测试连接后，即时更新状态；
3. 删除约束：已关联任务（未执行/执行中）的主机，不可删除，提示“该主机已关联未完成任务，请先终止或删除相关任务”；
4. 权限约束：普通运维仅能查看、编辑自己负责的主机，无法删除主机；高级运维可查看全量主机，编辑、删除非生产环境主机；管理员可操作全量主机；
5. 导入规则：导入时，若IP地址已存在，自动覆盖该主机的其他信息（除IP地址外）；若字段缺失（必填项），该条数据导入失败，不影响其他数据；
6. 负责人规则：主机负责人变更后，该主机的所有相关任务、日志，仍保留原负责人记录，新负责人可查看、操作该主机。

   - 支持按认证方式、连接状态筛选，支持按主机IP、用户名模糊搜索；
   - 鼠标悬停在认证方式上，显示具体认证类型（如“密码认证”“密钥认证”）。
2. 配置编辑交互：
   - 主机添加/编辑时，同步配置连接信息，选择认证方式后，动态显示对应字段（密码认证显示SSH密码；密钥认证显示SSH私钥、私钥密码）；
   - 输入SSH密码、私钥密码时，默认隐藏，可点击“显示密码”按钮切换显示/隐藏；

   - 点击“测试连接”按钮，系统立即尝试用当前配置连接主机，测试结果即时提示（“连接成功”/“连接失败：密码错误”/“连接失败：超时”）；
   - 保存配置后，密码、私钥、私钥密码自动加密存储，页面不显示明文。
3. 批量测试连接交互：
   - 主机列表、主机组详情页，支持勾选多台主机，点击“批量测试连接”；
   - 弹出测试进度弹窗，实时显示每台主机的测试状态（测试中/成功/失败）；
   - 测试完成后，提示“批量测试完成：成功XX台，失败XX台”，点击“查看详情”可查看失败原因；
   - 测试失败的主机，可点击“重新测试”，单独重新测试连接。

##### 4.1.3.4 业务规则

1. 关联规则：每台主机仅能有一个连接配置，添加主机时必须配置连接信息（认证方式、SSH用户名必填）；
2. 加密规则：SSH密码、私钥、私钥密码采用AES-256加密存储，数据库中不保存明文，页面展示时全部脱敏（显示为\*\*\*\*\*\*）；
3. 权限约束：普通运维仅能查看、编辑自己负责的主机的连接配置；高级运维可查看全量连接配置，编辑非生产环境主机的连接配置；管理员可操作全量连接配置；
4. 测试规则：系统每30分钟自动测试所有主机的连接配置，连接失败的主机状态改为“离线”，并记录失败原因；
5. 安全规则：修改连接配置后，系统自动测试连接，连接失败则不允许保存，提示“连接测试失败，请检查配置后重试”。

##### 4.2.1.3 交互说明

1. 脚本列表交互：
   - 列表展示脚本名称、类型、权限类型、作者、版本号、执行次数、成功率、审核状态，支持按这些字段筛选；

   - 草稿脚本标有“草稿”标识，待审核脚本标有“待审核”标识，审核拒绝脚本标有“审核拒绝”标识，高危脚本标有“高危”警示标识。
2. 脚本创建/编辑交互：

   - 点击“复制脚本”按钮，复制当前脚本的内容，生成新脚本（名称后缀加“-副本”），可修改后保存；
   - 编辑完成后，可选择权限类型：
     - 个人私有：仅自己可查看、编辑、执行；
     - 团队共享：选择可共享的团队/用户，被共享者可查看、执行，不可编辑；
     - 公共脚本：提交审核，管理员审核通过后，所有用户可查看、执行，不可编辑；
   - 点击“提交审核”（公共脚本），审核状态改为“待审核”，通知管理员审核；
   - 编辑过程中，点击“取消”，若有未保存内容，提示“是否放弃当前修改？”，确认后关闭编辑界面，草稿保留（若有）。

   - 点击“回滚到该版本”，弹出二次确认弹窗，提示“回滚后，当前脚本内容将替换为该版本，是否确认？”，确认后，版本号自动递增（如当前v1.0.2，回滚到v1.0.0，新版本为v1.0.3）；

   - 审核页面展示待审核脚本的名称、类型、作者、脚本内容、提交时间；

   - 审核过程中，点击“暂不审核”，审核状态保持“待审核”，后续可继续审核。

##### 4.2.1.4 业务规则

1. 唯一性规则：脚本名称唯一，不可重复创建（无论权限类型）；
2. 版本规则：每次修改脚本（除草稿外），版本号自动递增，保留所有历史版本，可回滚，不可删除历史版本；
3. 草稿规则：草稿脚本仅作者可查看、编辑，不参与执行、共享、审核；草稿保存时间无限制，作者可随时编辑、保存为正式脚本；
4. 权限规则：
   - 个人私有脚本：仅作者可查看、编辑、执行、删除；
   - 团队共享脚本：作者可查看、编辑、执行、删除、修改共享范围；被共享者可查看、执行，不可编辑、删除；
   - 公共脚本：作者可查看、编辑（编辑后需重新审核）、执行；所有用户可查看、执行；管理员可审核、删除；
5. 审核规则：公共脚本必须经过管理员审核通过后，才能进入公共脚本库；审核拒绝的脚本，作者需修改后重新提交审核；
6. 语法校验规则：系统仅做基础语法校验（如Shell脚本的语法错误、Python脚本的缩进错误），不校验脚本逻辑和执行结果；
7. 高危脚本规则：含重启、kill、rm、mkfs等危险命令的脚本，自动标记为“高危”，执行时需二次确认；
8. 删除规则：脚本删除后，所有历史版本、关联任务、执行日志仍保留，仅脚本本身不可再执行、编辑。

#### 4.2.2 任务定义与执行

##### 4.2.2.1 功能描述

定义脚本执行任务（关联脚本、目标主机、执行参数），支持多种执行方式（并发/串行/分批），支持Dry Run模拟执行、执行前预检查，确保脚本执行安全、高效。

##### 4.2.2.4 业务规则

1. 目标主机规则：
   - 任务目标主机必须至少1台，且均为在线状态（离线主机自动过滤，不参与执行）；
   - 若选择主机组，组内无在线主机，提示“所选主机组内无在线主机，无法执行任务”；
   - 执行过程中，主机变为离线，该主机执行失败，记录失败原因“主机离线”。
2. 执行参数规则：
   - 支持系统变量注入，变量自动替换为对应主机的信息（如`{{ip}}`替换为当前执行主机的IP）；
   - 参数填写错误（如变量拼写错误），执行时按原参数执行，不报错（脚本执行失败由脚本本身逻辑决定）；
   - 执行前预览功能仅展示参数替换后的命令，不校验命令逻辑。
3. 执行方式规则：
   - 并发执行：最大并发数不超过100台（避免服务器压力过大），超过100台自动转为分批执行（每批100台）；
   - 串行执行：前一台主机执行完成（成功/失败）后，再执行下一台；
   - 分批执行：每批执行完成后，间隔指定时间执行下一批，若某一批有主机执行失败，不影响下一批执行。
4. 超时规则：
   - 单台主机执行时间超过超时时间，自动终止该主机的执行，标记为“执行失败”，记录失败原因“执行超时”；
   - 任务总执行时间无限制，按单台主机超时时间分别判断。
5. 提权规则：
   - 提权时，sudo密码需正确，密码错误则该主机执行失败，记录失败原因“提权密码错误”；
   - 提权脚本执行时，自动在脚本前添加`sudo`命令，无需在脚本中手动添加。
6. 执行模式规则：
   - Dry Run模式仅检查脚本语法、参数替换、主机依赖，不实际在服务器上运行脚本，不影响服务器状态；
   - 静默执行不展示实时日志，仅在任务列表更新状态，执行完成后按配置发送通知，适合高频、无交互的巡检任务。
7. 任务状态规则：
   - 未执行：任务创建后，未点击“确认执行”；
   - 执行中：任务已开始执行，部分或全部主机未执行完成；
   - 执行成功：所有主机执行完成，且均执行成功；
   - 执行失败：至少有1台主机执行失败，其余主机可能成功/失败；
   - 已终止：执行过程中，用户手动终止任务；
   - 已暂停：执行过程中，用户手动暂停任务，可恢复执行。
8. 权限规则：
   - 普通运维仅能创建、执行、编辑、删除自己创建的测试环境任务；
   - 高级运维可创建、执行、编辑、删除自己创建的所有任务，可查看全量任务；
   - 管理员可操作全量任务，可强制终止任何执行中的任务；
   - 任务创建后，创建人可将任务转交他人（需对方有权限），转交后，任务负责人变为接收人。

   - 巡检项列表展示所有巡检项（默认项+自定义项），按巡检类别分组展示；
   - 默认巡检项标有「系统默认」标识，不可删除，仅支持编辑告警阈值、巡检脚本；
   - 自定义巡检项支持编辑、删除、复制，复制后生成新巡检项，可修改后保存；
   - 鼠标悬停巡检项名称，展示巡检脚本摘要、告警规则说明；
   - 支持按巡检类别、巡检项名称、告警级别筛选搜索。
2. 巡检任务创建交互：
   - 首页提供「一键系统巡检」快捷入口，点击直接进入快速创建页；
   - 系统默认勾选全部**系统默认巡检项**，用户可手动增减巡检项；
   - 目标主机支持选择主机组/多台主机，自动过滤离线主机；
   - 执行方式默认并发执行，可切换为分批执行并配置批次与间隔；
   - 支持配置执行模式：正常执行（展示实时日志）/静默执行（后台执行）；
   - 可配置巡检完成后自动发送报告至钉钉/企业微信/邮箱；
   - 点击「开始巡检」，高危巡检项自动弹出二次确认；生产环境巡检需验证码校验。
3. 巡检执行交互：
   - 执行页面展示整体巡检进度：总主机数、正常/警告/异常数量统计；
   - 按主机分组展示巡检结果，异常项自动标红置顶；
   - 单台主机巡检结果支持展开/收起，查看每项明细；
   - 巡检过程支持暂停、终止、重试异常主机；
   - 静默巡检仅在任务列表更新状态，不跳转日志页面。

##### 4.3.1.4 业务规则

1. 默认巡检项不可删除，仅允许管理员修改阈值与脚本；
2. 自定义巡检项脚本必须能输出可被平台解析的结构化结果（正常/警告/异常）；
3. 巡检任务执行时，单主机单巡检项超时不影响其他巡检项与主机；
4. 巡检结果按「异常 > 警告 > 正常」优先级排序展示；
5. 每日自动生成巡检历史记录，保留90天，超期自动归档压缩。

##### 拦截规则

## 5. 核心业务流程

### 5.1 日常一键巡检流程

1. 用户登录 → 首页点击「一键系统巡检」
2. 选择主机/主机组 → 确认巡检项
3. 系统自动过滤离线主机
4. 开始巡检 → 实时展示进度
5. 巡检完成 → 自动生成报告
6. 异常项标红展示 → 支持导出/通知
7. 记录审计日志

### 5.2 脚本批量执行流程

1. 进入脚本仓库 → 选择脚本 → 点击执行
2. 选择目标主机/组 → 填写参数
3. 配置执行方式/超时/提权
4. 预览最终执行命令
5. 高危/生产环境二次确认/验证码
6. 进入执行页面 → 实时日志输出
7. 执行完成 → 显示成功/失败统计
8. 支持重试失败机器 → 记录日志

### 5.3 故障排查WebSSH流程

1. 主机列表 → 选择异常主机
2. 点击「WebSSH」→ 建立终端连接
3. 在线执行命令排查问题
4. 操作全程录像审计
5. 退出关闭连接

### 5.4 公共脚本审核流程

1. 用户编辑脚本 → 提交公共审核
2. 管理员收到待审核通知
3. 查看脚本内容/危险命令扫描
4. 审核通过/驳回并填写意见
5. 通过则进入公共脚本库
6. 通知提交人结果

***

## 6. 异常处理规则

1. **主机连接失败**：自动标记离线，跳过执行，记录原因
2. **脚本执行超时**：自动终止，标记超时失败，不影响其他主机
3. **执行过程中断网**：任务暂停，重连后可继续
4. **脚本语法错误**：立即失败，返回错误行号
5. **权限不足**：拦截执行，提示无对应主机/脚本权限
6. **批量执行部分失败**：整体任务标记为“部分失败”，支持单独重试
7. **高危命令触发**：拦截或强制二次确认，不允许静默执行
8. **并发数超限**：自动转为分批执行，保证平台稳定

## 7. 验收标准

### 7.1 功能验收

- 主机增删改查、批量导入导出正常
- 脚本版本管理、草稿保存、语法高亮正常
- 批量执行、实时日志、重试、终止功能可用
- 一键巡检可正常执行并生成结构化报告
- 危险命令拦截生效，生产环境强校验
- 权限控制符合角色矩阵，越权操作被拦截
- 通知推送及时准确，内容脱敏
- 所有操作可审计，日志完整可追溯

### 7.2 体验验收

- 单任务创建配置≤1分钟完成
- 日志推送延迟≤1秒
- 列表加载≤2秒
- 高危操作有明显视觉提示
- 移动端可正常查看报告与日志

### 7.3 安全验收

- 密码、密钥加密存储，页面不展示明文
- 越权访问直接拦截并记录日志
- 高危操作必须二次确认或审批
- 不允许任何未日志化的后台操作
